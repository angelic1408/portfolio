Redesign the Portal Site

Role: Designer, Engineer

The site can be accessed here

Checkout LITE is an innovative payment service from Rakuten. It was developed on 2015 and now it has hundreds of merchants and still counting.

Problem

The desktop version existed before the mobile version, so it was hard to decide the breakpoint. Where to put all the menus and navigations in the mobile version.
The site analytics code (metrics) had to be changed to support the mobile version


Limitation

I was the sole engineer and developer for this project, and I have to communicate with other groups (analytics team and business team) by myself.


The Solution

Major changes that I made to the page wass to add a navigation menu when the page is viewed in mobile version.
I decided the break point by changing my screen smaller and smaller, and when the page breaks, I marked that size as my break point.


Learned Lesson

I managed to improved the UX score and page load speed (based on google page speed analytics).
The page is active now and listed as mobile-friendly by search engine. However, if I had the chance to improve, I'd like to optimize the image and assets (script and stylesheet).
