Redesign the Manual Site

Role: Designer, Engineer

The site can be accessed here

Checkout LITE Manual is a manual page, dedicated for developers who use Checkout LITE as one of their payment options. It has several pages including a API reference page (how to call the API), a dashboard page (how to login into the dashboard a.k.a your personal page), and features (such as Sandbox and Security Code).

Problem

The desktop version existed before the mobile version.
The Dashboard and API pages was structured in 3-coloumn-page. These two pages needed to be reconstructed almost entirely because I needed to change the grid to support the mobile version.
Designing the menu bar for mobile version.


Limitation

The entire project has to be done within a month. It consist of 10 subpages and the resources were limited.


Solutions

On mobile version, 2 menus are added on the left and right side to display primary and secondary navigation.
Reconstruct the grid, giving it a proper structure and class name. It's easier to maintain the code and it's reusable for the other pages too.


Learned Lesson

All the pages go live. Improve the UI/UX score up t0 100%, improve the page load speed by 20%.
If I have the chance, I'd like to make a uniform structure for all pages.
